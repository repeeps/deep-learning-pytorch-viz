PART 5 · ATTENTION

"배"의 정체를 밝혀라

같은 글자 배 는 과일🍐·신체🤰·선박🚢 중 무엇일까? 트랜스포머는 주변 단어를 얼마나 참고할지 스스로 계산해서 의미를 확정합니다. 아래 단계를 클릭하며 실제 숫자가 흐르는 걸 따라가 보세요.

초점 단어: 배 — 이 단어 하나가 업데이트되는 과정을 추적합니다.

Step 00 — Embedding

단어는 벡터로 시작하지만, 실제론 "한 덩어리(텐서)"로 들어간다

컴퓨터는 글자를 모릅니다. 각 단어(토큰)는 의미를 담은 숫자 묶음, 즉 임베딩 벡터가 됩니다(여기선 3차원). 그런데 모델에 넣을 땐 단어를 하나씩 넣지 않고, 벡터들을 쌓아 행렬/텐서 한 덩어리로 한꺼번에 넣어요. 토글로 두 관점을 비교해 보세요.

→ 이 시점의 배 벡터는 아직 "과일/신체/배" 중 무엇인지 모르는 중립 상태입니다.

Step 01 — Projection

하나의 단어가 Q · K · V 세 갈래로 분신한다

각 단어 벡터에 학습된 가중치 행렬(WQ, WK, WV)을 곱해 세 가지 역할의 벡터를 만듭니다. 행렬곱이 낯설다면 아래에서 출력 칸을 클릭해 계산이 분해되는 걸 보세요.

Query

"내가 찾는 것"

배가 주변에 던지는 질문. "나를 설명해 줄 단어 어디 있어?"

Key

"내가 가진 것"

각 단어가 들고 있는 명함/꼬리표. 질문과 맞춰보는 대상.

Value

"내가 줄 정보"

매칭되면 실제로 전달되는 내용물.

왜 굳이 단어 하나를 셋으로 쪼갤까?

같은 단어라도 역할이 달라서예요. 내가 물어볼 때(Query) 쓰는 정보와, 남이 나를 찾을 때 내미는 정보(Key), 막상 건네줄 알맹이(Value)가 서로 다르거든요. 임베딩 하나를 그냥 내적하면 이 세 역할이 뒤섞여요. 그래서 WQ·WK·WV 세 행렬로 같은 단어를 세 가지 모습으로 변신시켜, "질문↔명함"은 Key끼리, "전달"은 Value로 따로 쓰는 거예요.

Step 02 — Score · Scale · Softmax

"배"의 질문(Q)을 모든 명함(K)에 대본다

배의 Q 와 각 단어의 K 를 내적(dot product)하면 "얼마나 관련 있나" 점수가 나옵니다. → √d 로 나눠 안정화(scale) → Softmax 로 합이 1인 확률(비중)로 변환.

왜 √d로 나눌까(d = 벡터 차원): 차원이 크면 내적 점수가 너무 커지고, 큰 점수를 Softmax에 넣으면 한 단어에 비중이 99%+로 쏠려 나머지를 거의 못 보게 돼요(그러면 학습이 막혀요). √d로 나눠 점수 크기를 적당히 눌러, 비중이 골고루 퍼지게 만드는 안전장치예요. (지금 데모는 d=3이라 효과가 작지만, 실제 d=64·512에선 필수예요.)

❓ Softmax 가 뭐예요?

점수(아무 숫자나 가능)를 "합이 100%인 비중"으로 바꿔주는 변환입니다. 마치 점수에 비례해 주의(attention) 파이를 잘라 나눠주는 것과 같아요.

1 각 점수를 e^x (지수)로 — 전부 양수가 되고, 큰 값이 더 크게 벌어집니다 (1등을 부각).
2 각 e^x 를 전체 합으로 나눔 — 그러면 다 더해서 정확히 1(100%)이 됩니다.
softmax(xi) = exi ÷ ( ex1 + ex2 + … + exn )
① Q·K 점수 — 행을 클릭하면 내적 계산이 펼쳐집니다
② Softmax 변환 — 점수를 확률(비중)로

→ 배는 동사 먹었다 에 가장 큰 비중을 둡니다. 그래서 "먹는 대상 = 과일 배 🍐" 라는 해석이 강해져요.

Step 03 — Weighted Sum

비중만큼 Value를 섞는다 (비빔밥)

앞에서 구한 비중(softmax)을 각 단어의 Value 에 곱해 전부 더합니다. 비중 큰 재료가 맛을 지배하죠.

▶ 업데이트된 "배" 벡터 (context-aware)

→ 이제 배 벡터 안에 "먹는 행위"의 정보가 녹아들었습니다. 더 이상 중립이 아니라 과일 배 쪽으로 기울었어요.

Step 04 — Attention Map

전체 어텐션 히트맵

사실 모든 단어가 동시에 위 과정을 거칩니다. 각 행(Query)이 각 열(Key)을 얼마나 참고하는지를 한눈에 본 지도입니다. 행을 클릭해 보세요.

적게 참고많이 참고

→ 배 행을 보면 먹었다 칸이 가장 밝습니다 = 가장 많이 참고.

Step 05 — Layer Stacking

이 과정을 수십 번 층층이 반복한다

한 번으로 끝이 아닙니다. 같은 Attention 블록을 수십~수백 층 쌓아, 의미를 점점 정교하게 다듬습니다.

읽기 전 한 줄

아래 층별 역할 설명과 확률 숫자는 이해를 돕기 위한 비유·연출이에요. 실제 모델의 각 층이 "문법층/논리층/은유층"으로 이렇게 깔끔히 나뉘지는 않습니다(해석가능성 연구의 대략적 경향일 뿐). 핵심 메시지는 "같은 연산을 깊이 쌓을수록 표현이 정교해진다" 하나예요.

↑ 처음 들어온 데이터
Layer 1기초 문법·인접 단어 매칭 ("배 옆에 먹었다가 있네")
↑
Layer 12문장 논리·단어 간 관계 정립 ("배는 먹었다의 목적어")
↑
⋯ 수십 개 레이어 통과 ⋯
↑
Layer 96은유·상황 맥락 파악 ("철수가 허기를 달랜 과일")
↑ 고차원 추론 완성
Final"이 문맥의 배 = 먹는 과일 🍐"  (예시 확률 92/5/3% — 연출값)
🤔 왜 반복하나요?
  • 정제(Refinement): 얕은 층은 주변만 훑고, 깊은 층은 문장 전체의 숨은 의도까지 녹여 넣습니다.
  • 지능의 깊이: 층이 깊어질수록 텐서 속 숫자가 단순 값에서 추론 능력으로 진화합니다.
  • 병렬 전문가(Multi-Head): 각 층마다 여러 Head가 서로 다른 각도(문법/의미/위치…)에서 동시에 섞기를 수행합니다.
Step 06 — Playground

직접 입력하고 비중이 바뀌는 걸 보세요

문장을 띄어쓰기로 입력하면 토큰으로 쪼개집니다. 초점 단어(Query)를 고르면, 그 단어가 나머지 단어를 얼마나 참고하는지 비중이 실시간으로 다시 계산됩니다.

초점 단어 (Query) 선택 ▼
→ 배 가 각 단어를 참고하는 비중 (Attention Weights, 합=100%)
⚠️ 정직한 안내: 임베딩 벡터는 실제 학습된 값이 아니라 글자에서 결정론적으로 생성한 합성값입니다. 그래서 의미(과일/선박)를 진짜로 구분하진 못해요. 다만 내적 → √d 스케일 → Softmax 라는 어텐션 수식 자체는 100% 실제 계산이라, 토큰을 바꾸면 비중이 어떻게 출렁이는지 메커니즘을 체감할 수 있습니다.
