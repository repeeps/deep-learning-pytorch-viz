PART 3 · NLP BASICS

Word2Vec — 단어를 의미의 좌표로

컴퓨터는 단어를 숫자로만 다룹니다. 가장 단순한 방법인 원-핫 인코딩은 단어 사이의 의미·유사도를 전혀 담지 못해요. Word2Vec은 "비슷한 맥락에 등장하는 단어는 의미도 비슷하다"는 분포 가설을 이용해, 단어를 의미가 녹아든 조밀한 벡터(임베딩)로 학습합니다. 그러면 왕 - 남자 + 여자 ≈ 여왕 같은 벡터 산술까지 가능해져요.

Step 00 — One-hot Encoding

원-핫: 거대하고 텅 빈 벡터

어휘 사전에 단어가 V개 있으면, 각 단어는 자기 자리만 1이고 나머지는 전부 0인 V차원 벡터가 됩니다. 어휘가 수만 개면 벡터도 수만 차원이에요.

치명적 한계 — 모든 단어가 서로 직교

서로 다른 두 원-핫 벡터를 골라 내적(dot product)을 하면 항상 0입니다(같은 단어끼리만 1). 즉 "왕"과 "여왕"의 유사도나 "왕"과 "사과"의 유사도가 똑같이 0이에요. 의미가 전혀 담기지 않습니다.

Step 01 — Distributional Hypothesis

분포 가설 — 맥락이 의미를 만든다

"비슷한 맥락에 나오는 단어는 의미도 비슷하다." 그래서 Word2Vec은 문장 위에서 윈도우를 슬라이딩하며 (중심 단어, 주변 단어) 학습쌍을 뽑아냅니다. 중심 단어 기준 ±2칸을 윈도우로 봅니다.

이 위치에서 추출되는 학습쌍 (중심 → 주변)

→ 중심 단어와 윈도우 안의 주변 단어가 짝을 이뤄요. 윈도우를 끝까지 밀면 문장 전체에서 수많은 쌍이 나오고, 모델은 이 쌍들을 학습해 임베딩을 만듭니다.

Step 02 — CBOW vs Skip-gram

두 가지 학습 구조

같은 (중심, 주변) 쌍을 두고, 무엇으로 무엇을 예측하느냐가 다릅니다. CBOW는 주변으로 중심을, Skip-gram은 중심으로 주변을 예측해요.

CBOW (주변 → 중심)

여러 주변 단어를 평균해 가운데 빠진 중심 단어를 맞혀요. 학습이 빠르고 빈번한 단어에 유리해요.

Skip-gram (중심 → 주변)

하나의 중심 단어로 여러 주변 단어를 예측해요. 느리지만 적은 데이터·희귀 단어에서 표현이 더 좋아요.

그래서 "학습"이 뭔데? — 벡터를 밀고 당기기

핵심 직관 한 줄: 모델이 "이 중심 단어 옆엔 이 주변 단어가 온다"를 맞히도록 벡터를 조금씩 옮겨요. 같이 등장하는 단어쌍의 벡터는 서로 가깝게 당기고, 상관없는 단어는 밀어내죠. 이 밀당을 수많은 쌍에 반복하면, "비슷한 맥락의 단어"가 자연히 가까운 곳에 모입니다.

구체적으로는, 두 구조 모두 가운데 작은 은닉층(예: 300차원)을 통과하고, 학습이 끝나면 이 은닉층으로 가는 가중치 행렬의 각 행이 바로 우리가 원하는 단어 임베딩 벡터가 돼요. (아래 Step 3의 좌표는 그렇게 학습됐다고 가정한 예시값이에요.)

Step 03 — Embedding Space

의미가 좌표가 된 공간, 그리고 벡터 산술

학습이 끝나면 단어들은 의미에 따라 공간에 배치됩니다. 비슷한 단어는 가까이 모이고, 단어 사이의 방향(차이 벡터)이 일정한 의미를 가져요. 그래서 왕 - 남자 + 여자를 계산하면 여왕 근처에 도착합니다.

정직하게 — 좌표는 미리 학습된 값을 가정

실제 Word2Vec 학습은 거대한 말뭉치가 필요하므로 이 페이지에선 불가능해요. 그래서 아래 2D 좌표는 "이미 학습됐다고 가정한" 하드코딩 값입니다. 다만 A - B + C 벡터 산술과 그 최근접 단어 찾기는 JS로 실제 계산합니다.

산술: − + ≈

→ AB 를 빼고 C 를 더한 ★ 결과 지점에서 가장 가까운 실제 단어를 초록으로 강조했어요. 이게 바로 "유추(analogy)" 능력입니다.
