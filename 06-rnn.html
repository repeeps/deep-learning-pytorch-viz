PART 2 · ARCHITECTURES

RNN / LSTM — 기억을 다음 시점으로 흘려보내기

순환 신경망(RNN)의 핵심은 단 하나예요. 매 시점마다 같은 셀이 은닉 상태 h 를 다음 시점으로 넘겨준다는 것. 이 작은 고리 덕분에 순서가 있는 데이터를 다룰 수 있어요. 여기에 게이트를 달아 기억을 더 멀리 나르는 게 LSTM 입니다. 작은 숫자로 직접 굴려 봅시다.

Step 00 — Recurrence

한 셀이 자기 출력을 다시 입력으로 받는다

RNN 셀은 시점 t 의 입력 xt 와 직전 은닉 상태 ht-1 를 함께 받아 새 은닉 상태 ht 를 만들어요. 이 h 가 곧 지금까지의 기억입니다.

ht = tanh( Wx·xt + Wh·ht-1 + b )
고리(loop)의 의미

그림의 되먹임 화살표가 핵심이에요. ht 가 다음 칸으로 가서 ht-1 역할을 합니다. 그래서 셀은 딱 하나지만, 시점마다 같은 가중치로 계속 재사용돼요.

tanh 는 값을 -1 ~ 1 로 눌러 담아 상태가 폭주하지 않게 합니다.

→ 참고로 h는 보통 여러 개의 숫자로 된 벡터예요(여기 다음 스텝에선 2차원으로 봅니다). 다음 스텝에서 이 고리를 옆으로 쭉 펴서(unroll) 실제 숫자를 따라가 봐요.

Step 01 — Unroll Through Time

같은 셀을 시퀀스 길이만큼 옆으로 펴기

길이 4짜리 입력 시퀀스 x = [1.0, 0.5, -0.5, 1.5] 를 넣어 봅시다. h 는 2차원 벡터로 시작값은 [0, 0]. 버튼으로 시점을 진행하면 h 가 한 칸씩 실제로 갱신돼요.

하드코딩된 가중치 (2차원 은닉)

Wx = [0.8, -0.5]T  ·  Wh = [[0.5, -0.3],[0.2, 0.7]]  ·  b = [0.1, 0.0]

t = 0
이번 시점 계산 자세히

→ h 는 시점을 거치며 이전 기억 + 새 입력을 섞어 계속 갱신돼요. 마지막 h 하나에 시퀀스 전체 요약이 담깁니다.

Step 02 — LSTM Gates

게이트 3개로 기억(cell state)을 조절

LSTM 은 별도의 셀 상태 C(장기 기억)를 두고, 0~1 사이의 게이트로 "얼마나 버리고 / 받아들이고 / 내보낼지"를 정해요. 게이트를 직접 움직여 C 가 어떻게 바뀌는지 보세요.

C̃t(새 후보) = 지금 들어온 입력에서 만든 "이번에 새로 적어둘 만한 정보"예요(실제론 tanh가 만들지만 여기선 1.0으로 고정). Ct-1는 직전까지의 기억으로 2.0 고정. 게이트만 움직여 보세요.

forget gate f

이전 기억을 얼마나 남길까. 1이면 통째로 유지, 0이면 싹 잊음.

input gate i

새 후보 정보를 얼마나 받아들일까. 0이면 새 입력 무시.

output gate o

갱신된 기억 중 얼마를 밖으로 내보낼까. ht 를 결정.
0.90
0.60
0.50
게이트 값 — 0~1 (실제 LSTM에선 σ(시그모이드)가 만들지만, 여기선 직접 조절해 봅니다)
Ct = f · Ct-1 + i · C̃t   |   ht = o · tanh(Ct)

→ forget 을 높게 두면 옛 기억이 거의 그대로 흘러가요. 이게 LSTM이 장기 의존성을 잡는 비결입니다.

Step 03 — Why LSTM?

긴 시퀀스에서 옛 정보를 잃지 않으려면

기본 RNN 은 매 시점 가중치 |w|<1 을 곱하고 또 곱하기 때문에, 시점이 멀어질수록 초기 신호가 기하급수적으로 사라져요 — 학습 시 기울기도 같은 이유로 소실됩니다. LSTM 의 셀 상태는 1에 가까운 forget 게이트를 곱할 뿐이라 거듭 곱해도 거의 줄지 않고, 새 정보는 덧셈으로 더해져요.

0.70
기본 RNN

초기 신호 × |w|t. |w|<1 이면 시점마다 줄어들어 만 남아요. 옛 기억이 사라집니다.

LSTM

forget≈1 인 셀 상태는 거의 1 을 곱하므로 신호가 멀리까지 보존돼요. 유지.

→ 슬라이더를 내릴수록 빨간 곡선(RNN)이 더 빨리 0으로 꺼져요. 초록 곡선(LSTM)은 평평하게 정보를 지킵니다. |w|<1 을 거듭 곱하느냐, ≈1 을 곱하느냐의 차이예요.
