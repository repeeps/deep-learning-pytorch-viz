PART 6 · LLM

사전훈련 언어모델 — 먼저 언어를 배우고, 나중에 일을 배운다

트랜스포머(ch17) 위에 세워진 BERT와 GPT는 거대한 텍스트로 언어 자체를 미리 배워둡니다(사전훈련). 그다음 소량의 라벨 데이터로 원하는 작업에 적응시키죠(파인튜닝). 빈칸 맞추기와 다음 단어 잇기를 직접 눌러보면서 두 모델의 차이를 체감해 봅시다.

Step 00 — Pretrain → Fine-tune

두 단계 학습: 언어를 배우고 → 작업에 적응한다

예전에는 작업(감성분석, 번역…)마다 모델을 처음부터 학습했어요. 지금은 거대한 코퍼스로 언어 자체를 배운 모델을 가져와 소량의 라벨 데이터로 다듬기만 합니다.

비유: 외국어 원어민 채용 vs 신생아 교육

회사에 한국어 상담원이 필요하다고 해봅시다. 신생아를 데려와 한국어부터 가르치는 것(처음부터 학습)과, 이미 한국어가 유창한 사람을 뽑아 상담 업무만 가르치는 것(사전훈련 모델 + 파인튜닝). 어느 쪽이 빠를까요? 당연히 후자죠. 사전훈련 모델은 이미 문법·어휘·상식을 갖춘 "원어민"입니다.

1단계 · 사전훈련 (Pre-training)

  • 데이터: 위키·책·웹 수십억 단어 — 라벨 없음(비지도)
  • 과제: 빈칸 맞추기(BERT) / 다음 단어 예측(GPT)
  • 배우는 것: 문법, 어휘, 세상 지식
  • 비용: GPU 수백~수천 대, 수주~수개월
→

2단계 · 파인튜닝 (Fine-tuning)

  • 데이터: 라벨 달린 수백~수만 문장(지도)
  • 과제: 감성분석, NER, QA… 내 작업
  • 배우는 것: 작업 요령만 (언어는 이미 안다)
  • 비용: GPU 1대로 몇 시간이면 충분

데이터 양 비교 (라벨 작업이 비싼 게 핵심)

→ 라벨 없는 텍스트는 인터넷에 사실상 무한합니다. 사전훈련은 이 공짜 데이터에서 언어를 배우고, 비싼 라벨 데이터는 마지막 적응에만 아껴 씁니다. 이 분업이 사전훈련 패러다임의 본질이에요.

Step 01 — BERT · Masked LM

BERT: 문장에 구멍을 뚫고 빈칸을 맞춘다 (MLM)

BERT의 사전훈련 과제는 단어 일부를 [MASK]로 가리고 원래 단어를 맞추는 것. 정답을 맞추려면 앞뒤 문맥을 모두 읽어야 하므로, 풀다 보면 언어 이해력이 저절로 생깁니다.

[MASK] 위치 선택 — 어디를 가릴까요?

양방향(Bidirectional)이 BERT의 무기

빈칸 앞의 "나는"과 뒤의 "를 먹었다"를 동시에 봐야 "먹을 수 있는 것"으로 후보가 좁혀집니다. GPT처럼 왼쪽만 보면 "나는" 다음엔 무엇이든 올 수 있어서 훨씬 막막하죠. 그래서 BERT는 문장 이해(분류, 추출)에 강합니다.

덤으로 BERT는 NSP(Next Sentence Prediction: 두 문장이 실제로 이어지는 쌍인지 맞추기)도 함께 풀며 문장 사이의 관계 감각을 익혔어요.

Step 02 — GPT · Next Token

GPT: 다음 단어를 예측하고, 그 위에 또 잇는다 (자기회귀)

GPT의 사전훈련 과제는 단 하나 — "다음 단어는?". 예측한 단어를 문장에 붙이고 다시 다음 단어를 예측합니다. 후보 막대를 클릭해서 문장을 직접 이어가 보세요.

현재 문장 (자기회귀로 자라는 중)
다음 토큰 후보 — 클릭하면 문장에 붙어요
단방향(왼→오) + 자기회귀(Autoregressive)

GPT는 왼쪽(과거) 토큰만 보고 다음을 예측합니다. 미래를 못 보게 마스킹(causal mask)되어 있죠. 한 토큰을 뽑으면 그걸 입력에 붙여 또 예측 — 이 단순한 루프가 문장 생성의 전부입니다. ChatGPT가 글을 "한 글자씩" 출력하는 이유가 바로 이거예요.

BERT vs GPT 한눈 비교

항목BERTGPT
읽는 방향양방향 (앞뒤 다 본다)단방향 (왼쪽만 본다)

Step 03 — Fine-tuning

같은 몸통, 다른 머리 — 작업마다 헤드만 갈아 끼운다

파인튜닝은 사전훈련된 BERT 몸통 위에 작은 분류 헤드 하나를 얹고, 내 라벨 데이터로 살짝 더 학습하는 것. 문장 전체의 요약 벡터인 [CLS] 토큰 출력이 헤드의 입력이 됩니다.

[CLS]는 BERT가 입력 맨 앞에 항상 붙이는 특수 토큰이에요(classification의 약자). 어텐션을 거치며 문장 전체 정보가 이 자리에 모이도록 학습돼서, "문장 한 줄 요약 벡터"로 쓰여요. 그래서 분류할 땐 이 [CLS] 출력 하나만 꺼내 헤드에 넣어요.

새로 추가 (랜덤 초기화, 학습 O) 사전훈련 가중치 + 미세조정 (학습 O, 작게) 사전훈련 가중치 동결 (학습 X)

같은 몸통 + 다른 머리 = 다른 작업

감성분석

[CLS] 벡터 → 2-클래스 헤드 → 긍정/부정. "이 영화 최고" → 긍정 확률.

개체명 인식 (NER)

토큰마다 헤드를 달아 → 인명/지명/기관… 태그. "철수가 서울에" → 철수=인명, 서울=지명.

질의응답 (QA)

지문 토큰 위에 시작/끝 위치 헤드 → 정답 구간을 지문에서 짚어냄.

→ 세 작업 모두 몸통(BERT)은 동일하고 머리(헤드)만 다릅니다. 언어 이해라는 무거운 짐은 사전훈련이 이미 졌고, 헤드는 가볍게 "출력 형식"만 배우면 되니까요.
