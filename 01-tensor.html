PART 1 · FOUNDATION

텐서 조작 — 모양을 바꿔도 데이터는 그대로

딥러닝의 모든 데이터는 텐서(tensor), 즉 "여러 차원으로 쌓은 숫자 묶음"입니다. 학습은 이 텐서의 모양(shape)을 계속 바꿔가는 과정이에요. reshape·squeeze·브로드캐스팅이 실제로 숫자를 어떻게 옮기는지 눈으로 따라가 봅시다.

Step 00 — Rank / Dimension

스칼라 → 벡터 → 행렬 → 3D 텐서

차원(축)이 하나씩 늘어날 뿐, 본질은 같은 "숫자 묶음"입니다. 축 개수를 rank 라고 불러요.

→ 딥러닝에서 가장 흔한 건 3D~4D 텐서예요. 예: 이미지 배치 [배치, 채널, 높이, 너비].

Step 01 — reshape / view

원소 개수는 보존, 모양만 재배치

reshape 는 같은 12개 숫자를 다른 모양의 칸에 순서대로(행 우선) 다시 채워 넣을 뿐입니다. 곱이 12가 되는 모양이면 무엇이든 가능해요.

평탄화된 원본 (flatten) — 항상 동일
모양 변경 결과

→ 곱이 12가 안 되는 모양(예: 5×3)은 reshape 든 view 든 둘 다 불가능해요(원소 개수가 안 맞으니까). view 의 고유한 차이는 메모리를 복사하지 않고 같은 데이터를 다른 모양으로 본다는 것 — 대신 메모리가 연속(contiguous)이어야 해서 .t()(전치) 직후엔 view 가 실패할 수 있습니다.

Step 02 — squeeze / unsqueeze

크기 1짜리 축을 없애거나(squeeze) 끼워넣기(unsqueeze)

크기가 1인 축은 데이터 양에 영향이 없지만 모양에는 영향을 줍니다. 모델이 기대하는 모양을 맞출 때 자주 써요.

왜 필요할까?

예: 흑백 이미지 1장이 [28, 28] 인데, 모델은 [배치, 채널, 28, 28] 을 기대합니다. unsqueeze 로 앞에 크기 1 축 두 개를 끼워 [1, 1, 28, 28] 로 맞춥니다.

Step 03 — Broadcasting

모양이 달라도 자동으로 늘려서 연산

모양이 다른 두 텐서를 더할 때, 파이토치는 크기 1인 축을 자동으로 복제(stretch)해서 모양을 맞춥니다. 메모리는 안 늘리고 똑똑하게요.

결과 칸을 클릭 → 어떤 두 숫자가 더해졌는지 보기

→ [3×1] 은 열 방향으로, [1×3] 은 행 방향으로 늘어나 [3×3] 이 됩니다. 규칙: 각 축을 뒤에서부터 맞춰보고, 한쪽이 1이면 그쪽을 복제 (축 개수가 다르면 앞에 1을 채워 간주, 둘 다 1이 아닌데 다르면 에러).
