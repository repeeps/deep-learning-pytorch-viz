PART 1 · FOUNDATION

분류 — 직선 긋기에서 확률 말하기로

"합격이냐 불합격이냐"처럼 답이 범주인 문제는 직선 회귀로 풀리지 않아요. 점수를 시그모이드로 눌러 0~1 사이의 확률로 바꾸고(로지스틱 회귀), 손실은 크로스엔트로피로 재고, 클래스가 여러 개면 소프트맥스로 일반화합니다. 슬라이더를 직접 움직이며 왜 그래야 하는지 확인해 봐요.

Step 00 — Why not Regression?

회귀로는 분류가 안 된다

공부시간(x)에 따른 합격(1) / 불합격(0) 데이터 8개에 직선 y = wx + b 를 최소제곱으로 피팅해 봤어요. 두 가지가 망가집니다: 확률 범위를 벗어나고, 이상치 하나에 경계가 휘청거려요.

—
직선 피팅의 두 가지 문제

① 범위 이탈 — 직선은 x가 커지면 1을 넘고, 작아지면 0 밑으로 내려가요. "확률 130%"는 해석이 안 됩니다. (빨간 점선 구간)

② 이상치 민감 — 버튼을 눌러 보세요.

→ 필요한 건 아무리 큰 입력이 와도 0~1 사이에 머무는 함수예요. 다음 스텝의 시그모이드가 그 역할을 합니다.

Step 01 — Sigmoid

시그모이드로 확률화 — 로지스틱 회귀

점수 z = wx + b 를 그대로 쓰지 않고 시그모이드 σ(z) = 1/(1+e−z) 에 통과시키면 출력이 항상 0~1 사이의 확률 p 가 됩니다. p ≥ 0.5 면 합격으로 판정해요. w는 S곡선의 가파름, b는 좌우 위치를 정합니다.

p = σ(wx + b) = 1 / (1 + e−(wx+b))  ·  결정경계: wx + b = 0 → x* = −b/w

Step 02 — Cross-Entropy

왜 MSE 대신 크로스엔트로피(BCE)인가

분류에서는 손실로 이진 크로스엔트로피(BCE) 를 써요. 정답을 확신을 갖고 틀리면 벌점이 폭발하도록 설계된 함수입니다. MSE는 아무리 크게 틀려도 벌점이 최대 1에서 멈춰서, 시그모이드와 만나면 학습 신호가 약해져요.

BCE = −[ y·log p + (1−y)·log(1−p) ]

기호가 어렵게 보여도 괜찮아요. log는 "0에 가까워질수록 −∞로 곤두박질치는 함수"라고만 알면 충분해요 — 그래서 확신했는데 틀릴수록(맞을 확률 p가 0에 가까울수록) 벌점 −log p 가 폭발해요. 식을 외울 필요 없이, 아래 곡선이 말하는 결론만 보면 됩니다.

또 하나의 이유(심화): 시그모이드 + MSE 조합은 손실 지형이 비볼록(non-convex) 이라 경사하강법이 골짜기에 헤매기 쉽지만, 시그모이드 + BCE 는 볼록해서 안전하게 내려갈 수 있어요. ("볼록/비볼록"은 CH 05의 손실 지형에서 직접 굴려볼 수 있어요. 지금은 'BCE가 학습에 더 유리하다'만 기억하면 충분.)

Step 03 — Softmax

소프트맥스 — 다중분류로 일반화

클래스가 3개 이상이면 시그모이드 대신 소프트맥스를 써요. 각 클래스의 점수 z 를 e^z 로 키운 뒤 합으로 나누면, 전부 양수이고 합이 정확히 100%인 확률 분포가 됩니다. 점수 슬라이더를 움직여 보세요.

로지스틱 회귀의 일반화

클래스가 딱 2개일 때 softmax([z, 0]) 의 첫 확률은 e^z/(e^z+1) = σ(z) — 정확히 시그모이드가 됩니다. 즉 로지스틱 회귀는 2-클래스 소프트맥스 회귀의 특수한 경우예요. 검산: z=1.5 → softmax([1.5, 0])₀ = — = σ(1.5).

파이토치에선 nn.CrossEntropyLoss 가 softmax + log + NLL 을 한 번에 처리해서, 모델은 점수(logit)만 출력하면 됩니다.
